### Описание
Скрипт для проверки корректности номера кредитной карты по алгоритму Луна.  
Написан на bash.  
Работоспособность проверена на Ubuntu 24.04.3 + bash 5.2.21(1).

### Использование

1) Сохраните скрипты `credit_card_task.sh` и `credit_card_task_test.sh` в удобном месте.
2) Сделайте скрипты исполняемыми:  
`chmod +x credit_card_task.sh credit_card_task_test.sh`
3) Запустите скрипт с номером кредитной карты в качестве аргумента:  
`./credit_card_task.sh <16-значный номер карты>`
4) Также можно запустить скрипт с различными тестовыми данными:  
`./credit_card_task_test.sh`

### Примеры

```
./credit_card_task.sh 4111111111111111
# Номер карты валиден!
./credit_card_task.sh 1111111111111111
# Номер карты НЕ валиден!
./credit_card_task.sh 123456789012345a
# Неверный формат: номер карты должен содержать ровно 16 цифр
```

Больше примеров - в файле `credit_card_task_test.sh`.

### Алгоритм Луна

1) Удвоить значение каждой второй цифры, начиная с первой цифры слева.
2) Если после шага 1 в данной позиции получилось двузначное число, то сложить его цифры между собой (реализуется вычитанием 9).
3) Сложить результаты удвоения, полученные на этапе 2, и цифры исходного номера, не подвергшиеся удвоению.
4) Если последняя цифра равна нулю, то ошибок нет.

### Обработка ошибок

- Проверка аргументов: пользователь должен передать ровно один аргумент.
```
./credit_card_task.sh 1a 2b 3c
Использование: ./credit_card_task.sh <16-значный номер карты>
```

- Проверка формата номера карты: номер карты должен состоять ровно из 16 цифр.
```
./credit_card_task.sh 123456789012345a
Неверный формат: номер карты должен содержать ровно 16 цифр
```

- `set -euo pipefail` для строгого отслеживания ошибок.
